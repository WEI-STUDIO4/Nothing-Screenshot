# Nothing Screenshot 🈚

## 故事开始的地方

深夜，一场激战正酣。我在关键时刻按下截图键，准备永远保存那完美的一帧。

但系统弹窗先跳了出来：“截图已保存”。

弹窗消失时，完美的画面也消失了。那个瞬间，永远错过了。

我意识到：截图工具不该打扰瞬间。它应该在按下快捷键的瞬间完成一切，然后悄然退场，让体验继续。

于是有了 Nothing Screenshot——一个只为记录瞬间而存在的工具。

---

## 特性 ✨

### 🎯 **瞬间记录者**
- **零延迟操作**：按下快捷键即完成截图，无等待、无确认
- **无痕体验**：不弹窗、不提示、不打断当前操作
- **隐形存在**：启动后完全隐身系统托盘，专注你的沉浸体验

### ⚡ **沉浸式友好**
- **全局热键响应**：全屏应用、游戏、演示中均可使用
- **微光反馈**：可选的屏幕闪白效果，如快门般轻触
- **智能归档**：自动时间戳命名，整齐保存在专属文件夹

### 🎨 **极简哲学**
- 简约的深灰色界面，减少视觉负担
- 扁平化设计，专注功能本质
- "🈚"字符图标——存在但不彰显

## 快速开始 🚀

### 安装
```bash
pip install pyautogui keyboard pillow pystray pywin32
```

### 运行
```bash
python nothing_screenshot.py
```

### 使用
1. 运行后程序自动隐身系统托盘
2. 关键时刻按下 `Ctrl+Shift+S`
3. 屏幕微光一闪（可选）—— 瞬间已永恒
4. 继续你的体验，无任何打断

## 为何选择 Nothing Screenshot？ 🤔

### 场景对比

**传统工具：**
```
按下截图键 → 等待响应 → 选择区域 → 确认保存 → 收到提示 → 返回操作
```

**Nothing Screenshot：**
```
按下截图键 → 屏幕微闪 → 继续你的世界
                       ↳ 后台：文件已保存 (screenshot_20251205_234512.png)
```

### 为谁设计？
- **沉浸体验者**：不想被打断的玩家、影音爱好者
- **内容创作者**：需要频繁截图但保持心流状态
- **效率工作者**：讨厌上下文切换的专业人士
- **瞬间收藏家**：珍惜每一个不可复现的时刻

## 技术实现 🛠️

| 组件 | 作用 | 设计考虑 |
|------|------|----------|
| keyboard | 全局热键监听 | 确保全屏状态下100%响应 |
| pyautogui | 屏幕捕获 | 多显示器支持，稳定可靠 |
| pystray | 系统托盘集成 | 最小化视觉存在感 |
| tkinter | 配置界面 | 需要时才显现的极简设置 |

## 设计原则 🧠

### 1. 瞬间性原则
> 截图操作应在人类感知阈值内完成（<200ms）

### 2. 无痕原则  
> 使用后不留下任何操作痕迹或记忆负担

### 3. 可预测原则
> 每次行为完全一致，形成肌肉记忆

### 4. 可选反馈原则
- **开启闪白**：知道"已记录"
- **关闭闪白**：完全静默记录

## 配置说明 ⚙️

配置文件：`nothing_screenshot_config.json`

```json
{
  "hotkey": "ctrl+shift+s",
  "save_path": "C:\\Users\\用户名\\Pictures\\瞬间记录",
  "start_minimized": true,
  "flash_screen": true
}
```

### 推荐配置

**沉浸模式（默认）：**
```json
{
  "flash_screen": true,
  "start_minimized": true
}
```
- 轻微闪白作为心理确认
- 启动即隐身

**绝对静默模式：**
```json
{
  "flash_screen": false,
  "start_minimized": true
}
```
- 完全无感操作
- 仅通过文件系统确认
